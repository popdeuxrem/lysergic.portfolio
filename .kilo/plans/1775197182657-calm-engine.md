## PHASE 2: Create Immersive App Structure

### Approach: Standalone Package
Create `/immersive/` as self-contained app:

```
immersive/
├── package.json          # Independent or workspace
├── vite.config.js        #Bruno-style config
├── sources/              # Bruno's sources (copied)
│   ├── index.html
│   ├── index.js
│   ├── Game/            # 50+ modules
│   └── style/
├── public/               # Symlink to root public OR copy
└── dist/                # Separate output
```

### Implementation

**Task 2.1: Clone Bruno Reference**
```bash
git clone --depth 1 https://github.com/brunosimon/folio-2025 /tmp/bruno-folio
```

**Task 2.2: Copy Sources**
```bash
cp -r /tmp/bruno-folio/sources/* immersive/
cp /tmp/bruno-folio/vite.config.js immersive/
cp /tmp/bruno-folio/package.json immersive/
```

**Task 2.3: Configure Paths**
- Update `vite.config.js` root: `'.'` (not `'sources/'`)
- Update `publicDir:` to `../public` (use shared assets)
- Keep `index.html` at root

**Task 2.4: Install Dependencies**
```bash
cd immersive && npm install
```

## BLOCKERS / INCOMPATIBILITIES

1. **Three.js Version:** Bruno uses 0.183+ with WebGPU
2. **Rapier:** Uses `@dimforge/rapier3d` (not -compat)
3. **CSS:** Bruno uses Stylus, need conversion
4. **Complexity:** 50+ modules to maintain separately

---

## UPSTREAM ARCHITECTURE REQUIREMENTS (For Phase 2)

When we proceed to port, we need to inventory:
- Entry points: `sources/index.js`, `sources/index.html`
- Runtime systems: `sources/Game/` (50+ modules)
- Asset directories: `static/vehicle/`, `static/areas/`, `static/basis/`, `static/draco/`
- Loaders: GLTF, KTX2, Draco in `ResourcesLoader.js`
- Physics: `@dimforge/rapier3d` via dynamic import
- Input: Keyboard/touch in `Inputs/`
- Camera: `View.js`
- Zones: `Zones.js`
